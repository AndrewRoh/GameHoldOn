# GDD — Office Layoff Survivor (working title)

인디·빠른 개발용 **MVP 초안**. 뱀파이어 서바이벌류 **2D 탑다운 자동 공격 서바이벌** + **IT 구조조정 풍자 서사**.

## 1. Overview

AI 도입·구조조정으로 흔들리는 회사에서, 플레이어는 **현장 개발자**로서 매주 살아남는다. **인사과장·대표·기술이사**가 상징하는 압박이 적(몹)으로 쏟아지고, 시간이 지날수록 **주차(Week)** 가 진행되며 난이도가 올라간다. **2달(8주)** 를 버티면 1라운드 클리어를 목표로 한다.

## 2. Player Fantasy

- **억울함을 힘으로**: 말은 통하지 않는다. 숫자·출근·OKR 대신 **화력과 생존**으로 한 주를 연명한다.
- **동료 개발자 연대감**(후속 버전): 픽업·버프로 확장 가능. 프로토타입에서는 1인 서바이벌에 집중.

## 3. Detailed Rules (MVP)

- **카메라**: 탑다운 2D, 플레이어 추적.
- **이동**: WASD (또는 방향키 — 입력맵 확장 예정).
- **자동 공격**: 가장 가까운 적을 향해 투사체 발사(쿨다운). 탄은 적과 충돌 시 피해.
- **적**: **HR(인사과장)**, **CEO(대표)**, **CTO(기술이사)** 3종. 체력·속도·스폰 가중치가 다름.
- **주차 진행**: **1주 = 한 스테이지 구간**. 구간 시간이 끝나면 `Week + 1`. **8주** 도달 시 **승리**(인디 MVP 기준).
- **패배**: 플레이어 HP 0.
- **난이도 스케일**: 주가 올라갈수록 스폰 주기 감소·적 체력/속도 소폭 상승.

## 4. Formulas (MVP tuning knobs)

| 변수 | 초안 |
|------|------|
| 주 길이 `WeekDurationSec` | 45 (프로토타입 압축; 본편에서 조정) |
| 스폰 간격 | `max(0.55, 2.4 - (Week-1)*0.22)` 초 |
| 주차별 적 HP 배율 | `1 + (Week-1)*0.12` |
| 주차별 적 속도 배율 | `1 + (Week-1)*0.08` |
| 플레이어 이동 속도 | 220 px/s |
| 투사체 DPS 느낌 | 발사 간격 0.28s, 명중 시 고정 피해(종별 보정 가능) |

## 5. Edge Cases

- 적이 없을 때: 투사체 발사 **쿨만 돌고 발사 안 함** 또는 마지막 방향 유지(후자는 추후).
- 8주 클리어 직후: **승리 UI** 후 재시작 또는 로비(후속).
- 웹 빌드: Godot C#은 **웹 미지원** — 데스크톱 타깃 전제.

## 6. Dependencies

- Godot 4.x .NET, `net8.0`.
- 아트: **PixelLab MCP** 캐릭터·탄환 (`design/quick-specs/pixel-art-mvp-2026-05-15.md`). 바닥 타일은 생성 대기 중이면 스크립트 플레이스홀더.
- 사운드: 미착수.

## 7. Tuning Knobs

- `WeekDurationSec`, 스폰 곡선, 3종 적 비율(HR:CEO:CTO), 플레이어 HP, 투사체 피해·속도.

## 8. Acceptance Criteria (prototype)

- [ ] `main.tscn` 실행만으로 **이동 + 자동 사격 + 적 스폰**이 된다.
- [ ] **Week 1→8** 텍스트가 구간마다 진행한다.
- [ ] **승리/패배** 상태가 화면에 표시된다.
- [ ] `dotnet build` 성공 유지.

## 9. Prologue (프롤로그)

> 음악가는 음악을 만들고, 시인은 시를 쓴다. 게임 개발자는 게임으로 자신을 표현하는 게 맞다고, 문득 생각이 들었다.

- **톤**: 조용한 독백 → 타이틀/첫 주 전 **짧은 텍스트**(스킵 가능)로 넣기 좋다.
- **연출(후속)**: 배경만·타자기·로고 페이드 등 인디 범위에서 선택.

- **CEO**: 방향 전환·"AI 퍼스트" 슬로건의 압박.
- **CTO**: 기술 부채·마이그레이션·야근을 정당화하는 압박.
