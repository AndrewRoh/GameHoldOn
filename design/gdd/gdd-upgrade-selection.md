# GDD — Upgrade Selection (레벨업 3선택 카드)

> Parent: `gdd-meta-progression.md` · Implementation: `src/UpgradeManager.cs` (신규), `src/Player.cs`, `src/GameBalance.cs`

## 1. Overview

플레이어가 레벨업할 때마다 게임이 **일시 정지**되고 무작위 **3장의 업그레이드 카드**가 제시된다.
플레이어는 그 중 하나를 선택하여 영구 스탯 향상 또는 신규 능력을 획득한다.
총 15종 카드(Common 7 · Rare 5 · Epic 3)로 구성되며, 레어도별 추첨 가중치와 레벨 해금 조건으로 조합의 다양성을 보장한다.
기존 자동 스탯 강화(`FireInterval(level)` · `DamageMultiplier(level)` 자동 적용)는 이 시스템으로 **대체**된다.

## 2. Player Fantasy

- "내 빌드를 내가 고른다" — 같은 플레이라도 선택에 따라 다른 강점을 키운다.
- 3초의 선택이 이후 흐름을 바꾸는 순간적 전략감.
- IT 풍자 테마의 카드 이름이 웃음을 유발하면서도 직관적 효과를 전달한다.

## 3. Detailed Rules

### 3-1. 레벨업 트리거

- 레벨업 XP 임계치 도달(기존 `gdd-meta-progression.md §4` 공식 유지).
- 레벨업 순간 **`Engine.TimeScale = 0`** (게임 완전 정지).
- 선택 UI(카드 3장) 표시 → 플레이어 선택 → `Engine.TimeScale = 1` 복원.
- 최고 레벨(Lv 8) 도달 후에는 선택 UI 없음.

### 3-2. 카드 추첨 규칙

| 단계 | 규칙 |
|------|------|
| 레어도 결정 | 아래 가중치 표에서 3회 독립 추첨 (중복 레어도 허용) |
| 카드 결정 | 해당 레어도 풀에서 **해금 조건 충족 카드** 중 무작위 1장 |
| 중복 카드 | 동일 카드 3장 중복 방지 — 이미 뽑힌 카드는 재뽑기 |
| 풀 소진 시 | 레어도 한 단계 낮은 풀에서 보충 |

**레어도 가중치:**

| 레어도 | Lv 2–4 | Lv 5–6 | Lv 7–8 |
|--------|--------|--------|--------|
| Common | 80 | 65 | 50 |
| Rare | 20 | 30 | 38 |
| Epic | 0 | 5 | 12 |

### 3-3. 카드 효과 적용

- 선택 즉시 **누적(스택) 방식**으로 Player에 영구 적용.
- 각 카드는 개별 최대 스택 수 지정; 최대 도달 시 해당 카드는 추첨 풀에서 제거.
- 기존 `FireInterval(level)` / `DamageMultiplier(level)` 자동 공식은 **비활성화** — 발사 간격·피해는 카드 효과로만 변동.

### 3-4. 입력

| 입력 | 행동 |
|------|------|
| 마우스 좌클릭 | 해당 카드 선택 |
| 키 `1` / `2` / `3` | 왼쪽·가운데·오른쪽 카드 선택 |
| ESC / 일시정지 | 무시 (선택 강제) |

## 4. 카드 풀 & 수치 정의

### 4-1. Common 카드 (기본 추첨 풀, Lv 2+)

| # | 카드명 | 효과 | 최대 스택 | 스택 누적 공식 |
|---|--------|------|-----------|----------------|
| C1 | **야근 특근** | 발사 간격 ×0.87 | 4 | `FireCD × 0.87^n` → 최대 ×0.573 |
| C2 | **커밋 러시** | 투사체 +1 | 3 | `ProjectileCount + n` → 최대 4발 |
| C3 | **풀리퀘 머지** | 피해 ×1.18 | 4 | `DmgMult × 1.18^n` → 최대 ×1.939 |
| C4 | **핫픽스** | 탄속 ×1.25 | 2 | `ProjSpeed × 1.25^n` → 최대 ×1.5625 |
| C5 | **린 방법론** | 이동속도 ×1.10 | 3 | `MoveSpeed × 1.10^n` → 최대 ×1.331 |
| C6 | **스프린트 가속** | XP 획득 ×1.30 | 2 | `XpGain × 1.30^n` → 최대 ×1.69 |
| C7 | **연차 사용** | 최대 HP +30, 즉시 회복 +30 | 4 | `MaxHp + 30n` → 최대 +120 |

|---|--------|------|-----------|------|

| R2 | **코드 리뷰** | 치명타율 +15% | 3 | 최대 45%; 치명타 피해 ×1.5 |

| R4 | **재택근무** | 접촉 피해 회피율 +10% | 2 | 최대 20%; 각 틱 독립 판정 |

### 4-3. Epic 카드 (Lv 5+)

| # | 카드명 | 효과 | 최대 스택 | 비고 |
|---|--------|------|-----------|------|
| E1 | **구조조정 역습** | 투사체 +2, 피해 ×1.40 | 1 | 한 번만 획득 가능 |
| E2 | **주니어 개발자** | 궤도 투사체 +1 | 2 | 반경 80px 공전; 0.8s마다 기본 피해 |
| E3 | **CEO 아이디어** | 피해·발사속도·이동속도 각 ×1.15 | 1 | "랜덤한 올 버프" — 한 번만 획득 |

### 4-4. 궤도 투사체 (E2 전용)

- 플레이어 중심으로 반경 80px 공전, 각속도 150°/s.
- 0.8s 간격으로 가장 가까운 적에게 `ProjectileBaseDamage × 0.8` 피해 (별도 DmgMult 미적용).
- 스택 2시: 두 번째 궤도체는 반경 100px, 180° 위상차.

## 5. Edge Cases

| 상황 | 처리 |
|------|------|
| 추첨 풀이 3장 미만 | 남은 카드 전부 표시; 빈 슬롯에 "기운 회복 (HP +15)" 항상 추가 |
| 레벨업과 동시에 사망 | 사망 우선 처리 — 선택 UI 열지 않음 |
| Week 전환 중 레벨업 | 페이드아웃 종료 후 선택 UI 표시 |
| 야근 특근 4스택 이미 보유 | 해당 카드 풀에서 제거 → 추첨 다시 |
| C2(커밋 러시) 3스택 = 4발 시 투사체 방향 | 메인 1발 + 좌우 ±12° 보조 발사 |

## 8. Acceptance Criteria

- [ ] 레벨업 시 `Engine.TimeScale = 0` → 카드 3장 UI 표시 → 선택 후 `TimeScale = 1` 복원.
- [ ] 마우스 클릭 / 키 `1·2·3`으로 선택 가능, 선택 즉시 스탯 반영.
- [ ] Lv 2–4에서 Epic 카드(E1·E2·E3)가 등장하지 않는다.
- [ ] 동일 카드 3장 동시 제시 불가 (중복 방지 로직 동작).
- [ ] C2(커밋 러시) 3스택: 투사체 4발이 ±12° 각도로 발사된다.
- [ ] E2(주니어 개발자) 2스택: 공전 투사체 2개가 180° 위상차로 존재한다.
- [ ] 최대 스택 도달 카드는 이후 추첨에서 등장하지 않는다.
- [ ] 카드 적용 후 HUD 스탯이 실시간 갱신된다.
- [ ] Lv 8 도달 이후 레벨업 이벤트가 발생하지 않는다.
